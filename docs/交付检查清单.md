# 交付检查清单

本文档用于每次省份 Mock 演示交付前检查。交付前建议逐项确认。

## 基础检查

- [ ] 已确认项目定位为前端 Mock 验收演示系统。
- [ ] 已保留原始省份评估 Excel。
- [ ] 已确认 Excel 中存在 `2、功能点拆分表`。
- [ ] 已生成或更新省份页面。
- [ ] 已生成或更新省份路由。
- [ ] 已补充省份演示清单或验收说明。

## 页面检查

- [ ] 省份一级入口可见。
- [ ] 功能用户菜单可展开。
- [ ] 功能用户需求页面可打开。
- [ ] 页面无空白页。
- [ ] 表格有模拟数据。
- [ ] 查询和重置按钮有反馈。
- [ ] 新增、编辑、详情、删除等基础操作可点击。
- [ ] 导入、导出按钮有反馈。
- [ ] 审批、下发、反馈、稽核、统计类按钮能打开对应弹窗。
- [ ] 页面刷新后不出现明显报错。

## 流程检查

- [ ] 审批类流程能展示节点、处理人、意见和结果。
- [ ] 接口调用类流程能展示请求、响应和调用日志。
- [ ] 稽核校验类流程能展示通过项和异常项。
- [ ] 统计分析类流程能展示图表或汇总信息。
- [ ] 日志类流程能展示操作时间线。
- [ ] 找不到明确类型的功能过程已使用通用操作弹窗承接。

## 构建检查

执行：

```bash
npm run build
```

确认：

- [ ] 构建成功。
- [ ] `dist/` 已生成。
- [ ] 构建过程中无阻断性错误。
- [ ] 如有 warning，已确认不影响演示。

## 演示检查

- [ ] 已准备推荐演示路径。
- [ ] 已标记重点页面。
- [ ] 已准备 Mock 口径说明。
- [ ] 已确认演示账号、访问地址和启动命令。
- [ ] 已确认现场不依赖真实后端服务。

## 交付内容

建议交付或保留：

- [ ] 源码工程。
- [ ] 原始 Excel。
- [ ] 生成脚本。
- [ ] 生成清单。
- [ ] 省份演示清单。
- [ ] 构建产物 `dist/`。

## Mock 口径

验收说明中应明确：

- 本系统为前端 Mock 演示。
- 页面数据为模拟数据。
- 导入、导出、审批、接口调用、稽核、统计等为前端模拟效果。
- 系统用于功能点覆盖演示，不代表生产环境最终实现。
