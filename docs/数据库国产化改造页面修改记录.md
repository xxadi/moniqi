# 数据库国产化改造页面修改记录

## 概述

基于 `src/data/功能点备注.json` 中"数据库国产化改造"模块的 248 条记录，自动生成 50 个 Vue 页面。

- 生成脚本：`scripts/generate-db-migration.js`
- 页面目录：`src/views/资产信息上报调整/数据库国产化改造/{一级功能}/index.vue`
- 路由文件：`src/router/modules/examples.js`（资产信息上报调整 > 数据库国产化改造）

## 修改内容

### 1. 按钮操作类型区分（getActionType 匹配顺序）

**问题**：多个按钮匹配到相同操作类型，弹窗逻辑相同。

**修复**：调整 `getActionType` 函数中正则匹配顺序，确保：
- "录入"在"调度"之前（"录入调度..."按钮主操作是录入）
- "调用"在"调度"之前（"调用...模块...流程"包含"调度"但主操作是调用）
- "反馈"在"调度"之前（"反馈...模块...结果"包含"调度"但主操作是反馈）
- "调度"使用更精确的正则，避免误匹配模块名中的"任务调度"

```js
function getActionType(name) {
  // 更细粒度的分类，避免同类按钮过多
  // 接收系列优先
  if (/接收.*请求|接收.*处理/.test(name)) return "receive_request";
  if (/接收.*反馈|接收.*信息|接收.*结果/.test(name)) return "receive_feedback";
  if (/接收/.test(name)) return "receive";
  if (/发送/.test(name)) return "send";
  // 录入 必须在 调度 之前，因为 "录入调度..." 按钮主操作是录入
  if (/录入|配置|新增|创建|保存|建立|修改/.test(name)) return "configure";
  // 调用 必须在 调度 之前，因为 "调用...模块...流程" 包含"调度"但主操作是调用
  if (/调用.*流程|调用.*处理/.test(name)) return "invoke";
  if (/调用/.test(name)) return "send";
  // 反馈 必须在 调度 之前，因为 "反馈...模块...结果" 包含"调度"但主操作是反馈
  if (/反馈.*结果|返回.*结果/.test(name)) return "feedback_result";
  if (/反馈|返回|输出/.test(name)) return "feedback";
  // 调度 放在最后，避免误匹配模块名中的"任务调度"
  if (/^调度|唤醒/.test(name)) return "dispatch";
  if (/调度(?!模块)/.test(name)) return "dispatch";
  if (/查询|读取|展示|监控|跟踪/.test(name)) return "query";
  if (/导出|下载/.test(name)) return "export";
  if (/删除|移除/.test(name)) return "delete";
  if (/分析|统计|报表/.test(name)) return "analysis";
  return "operation";
}
```

**关键**：`getButtonIcon` 函数也要同步调整顺序。

### 2. 录入弹窗字段与列表一致

**问题**：录入弹窗是固定字段（配置名称、配置类型等），与表格列不匹配。

**修复**：弹窗使用 `editableColumns` 动态渲染，与表格列完全一致：

```html
<!-- 录入/配置弹窗 - 字段与列表一致 -->
<el-dialog :title="configureTitle" :visible.sync="configureVisible" width="620px" append-to-body>
  <el-form :model="configureForm" label-width="130px">
    <el-form-item v-for="col in editableColumns" :key="col.prop" :label="col.label">
      <el-select v-if="isStatusField(col.prop)" v-model="configureForm[col.prop]" style="width: 100%">
        <el-option label="待处理" value="待处理"></el-option>
        <el-option label="处理中" value="处理中"></el-option>
        <el-option label="已完成" value="已完成"></el-option>
        <el-option label="处理失败" value="处理失败"></el-option>
        <el-option label="已暂停" value="已暂停"></el-option>
        <el-option label="异常" value="异常"></el-option>
      </el-select>
      <el-input-number v-else-if="isNumberField(col.prop)" v-model="configureForm[col.prop]" :min="0" controls-position="right" style="width: 100%"></el-input-number>
      <el-input v-else-if="isLongField(col.prop)" v-model="configureForm[col.prop]" type="textarea" :rows="3"></el-input>
      <el-input v-else v-model="configureForm[col.prop]"></el-input>
    </el-form-item>
  </el-form>
  <span slot="footer">
    <el-button @click="configureVisible = false">取消</el-button>
    <el-button type="primary" @click="submitConfigure">保存配置</el-button>
  </span>
</el-dialog>
```

### 3. openConfigure 初始化表单

```js
openConfigure(functionName) {
  this.configureTitle = functionName;
  const S = pageHash(functionName);
  const row = this.createEmptyRow();
  const now = this.now();
  const primary = this.getPrimaryNameField();
  const status = this.getStatusField();
  if (primary) row[primary.prop] = functionName;
  if (status) row[status.prop] = "待处理";
  // 填充其他可编辑字段
  this.editableColumns.forEach((col, i) => {
    if (primary && col.prop === primary.prop) return;
    if (status && col.prop === status.prop) return;
    if (/线程/.test(col.prop)) row[col.prop] = pick([4, 8, 16, 32, 64], i, S);
    else if (/负载/.test(col.prop)) row[col.prop] = pick(["轮询", "权重", "随机", "最少连接"], i, S);
    else if (/耗时/.test(col.prop)) row[col.prop] = 0;           // 耗时默认0
    else if (/时间/.test(col.prop)) row[col.prop] = now;         // 时间用 this.now()
    else if (/编号|ID/i.test(col.prop)) row[col.prop] = "";
    else row[col.prop] = pick(["正常", "运行中", "已完成", "待处理"], i, S);
  });
  this.configureForm = row;
  this.configureVisible = true;
},
```

### 4. submitConfigure 添加数据到列表

```js
submitConfigure() {
  const now = this.now();   // 统一用 this.now() 保证格式一致
  const id = "CFG-" + Date.now();
  const primary = this.getPrimaryNameField();
  const newRow = {
    ...this.configureForm,
    "ID": id,
  };
  // 如果有编号字段，自动填充
  if (/编号|ID/i.test(primary ? primary.prop : "")) {
    newRow[primary.prop] = id;
  }
  // 更新时间设为当前时间
  const timeCol = this.editableColumns.find(c => /时间/.test(c.prop));
  if (timeCol) newRow[timeCol.prop] = now;
  this.allTableData.unshift(newRow);
  this.pageTotal = this.allTableData.length;
  this.fetchData();
  const name = primary ? newRow[primary.prop] : id;
  this.$message.success("配置已保存并添加到列表：" + name);
  this.configureVisible = false;
},
```

### 5. 时间格式统一

**问题**：`toLocaleString("zh-CN")` 输出格式与 mock 数据不一致。

**修复**：全部使用 `this.now()` 方法，输出 `YYYY-MM-DD HH:mm:ss` 格式：

```js
now() {
  const d = new Date();
  const pad = (n) => String(n).padStart(2, "0");
  return `${d.getFullYear()}-${pad(d.getMonth() + 1)}-${pad(d.getDate())} ${pad(d.getHours())}:${pad(d.getMinutes())}:${pad(d.getSeconds())}`;
},
```

### 6. 操作类型与弹窗映射

| 操作类型 | 正则匹配 | 弹窗类型 |
|---------|---------|---------|
| receive_request | 接收.*请求\|接收.*处理 | 表单录入（字段与列表一致，保存后添加到列表） |
| receive_feedback | 接收.*反馈\|接收.*信息\|接收.*结果 | 数据流查看面板（字段预览表格） |
| receive | 接收 | 线程处理详情 |
| send | 发送 | 调度详情 |
| configure | 录入\|配置\|新增\|创建\|保存\|建立\|修改 | 表单录入（字段与列表一致） |
| dispatch | 调度\|唤醒 | 调度面板（节点状态+策略） |
| invoke | 调用.*流程\|调用.*处理 | 流程执行面板（分步流水线） |
| feedback_result | 反馈.*结果\|返回.*结果 | 结果反馈详情（成功/失败告警） |
| feedback | 反馈\|返回\|输出 | 反馈结果面板 |
| query | 查询\|读取\|展示\|监控\|跟踪 | 查询面板 |
| export | 导出\|下载 | 导出对话框 |
| delete | 删除\|移除 | 删除确认 |
| analysis | 分析\|统计\|报表 | 业务分析面板 |
| operation | 其他 | 通用操作表单 |

## 注意事项

1. **正则匹配顺序很重要**：
   - `录入` 必须在 `调度` 之前，否则"录入调度..."按钮会错误匹配为 dispatch
   - `调用` 必须在 `调度` 之前，否则"调用...模块...流程"会错误匹配为 dispatch
   - `反馈` 必须在 `调度` 之前，否则"反馈...模块...结果"会错误匹配为 dispatch
2. **调度正则要精确**：使用 `^调度|唤醒` 或 `调度(?!模块)` 避免误匹配模块名中的"任务调度"
3. **getButtonIcon 要同步调整**：图标映射的正则顺序必须与 getActionType 一致
4. **时间格式统一用 `this.now()`**：不要用 `toLocaleString()`，格式不一致
5. **耗时字段默认值为 0**：新录入的数据耗时初始为 0
6. **录入后数据添加到列表顶部**：使用 `unshift` 而非 `push`
