# 省份接入流程

本文档用于规范新增省份功能点评估表到 Mock 演示系统的操作流程。

## 1. 准备输入文件

将省份评估结果 Excel 放入项目根目录或约定输入目录。文件名建议保留省份、年份、项目名称和版本号。

示例：

```text
中国移动云南公司2024年资产安全管理平台研发项目-评估结果(1).xlsx
```

接入前确认 Excel 包含 `2、功能点拆分表`，并检查表头是否包含：

- 客户需求
- 功能用户
- 功能用户需求
- 触发事件
- 功能过程
- 子过程描述
- 数据移动类型
- 数据组
- 数据属性

## 2. 解析功能点结构

解析 `2、功能点拆分表`，统计以下信息：

- 客户需求数量
- 功能用户数量
- 功能用户需求数量
- 功能过程数量
- 子过程描述数量

推荐生成一份解析摘要，用于判断页面规模和演示重点。

## 3. 确认菜单规划

标准菜单层级为：

```text
资产信息上报调整
└── {省份模块}
    └── {功能用户}
        └── {功能用户需求页面}
```

如果某个省份页面过多，可以将功能用户需求数量较大的模块放在前面，作为重点演示模块。

## 4. 生成页面

页面生成规则：

- 功能用户需求生成一个页面。
- 功能过程生成页面内按钮、表格操作或流程入口。
- 子过程描述生成弹窗步骤、流程日志、接口日志或详情说明。
- 数据组和数据属性用于推导表格字段和表单字段。

生成输出建议：

```text
src/views/资产信息上报调整/{省份模块}/
src/router/modules/examples.js
docs/{省份模块}页面生成清单.md
```

## 5. 生成后检查

生成后至少检查：

- 省份菜单是否出现。
- 每个功能用户下是否有页面。
- 页面是否能打开。
- 页面表格是否有数据。
- 功能过程按钮是否生成。
- 弹窗是否能打开。
- 浏览器控制台是否无明显报错。

## 6. 构建验证

执行：

```bash
npm run build
```

如果构建失败，优先检查：

- 路由路径是否重复。
- Vue 组件 `name` 是否重复或非法。
- 中文目录路径是否被错误转义。
- 生成的字符串是否缺少引号或转义。
- 自动生成页面是否存在语法错误。

## 7. 编写演示说明

每个省份建议补充一份演示说明：

```text
docs/{省份模块}演示清单.md
```

内容包括：

- 省份模块入口路径。
- 重点演示页面。
- 重点按钮和弹窗。
- 需要说明的 Mock 边界。
- 验收时建议点击路径。

## 8. 交付前归档

交付前确认以下文件已保留：

- 原始 Excel。
- 页面生成脚本。
- 生成清单。
- 演示清单。
- 验收检查结果。

不建议只交付 `dist`，否则后续无法追溯页面来源和功能点覆盖情况。
