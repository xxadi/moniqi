# 项目标准

## 项目定位

本项目为资产安全管理平台功能点验收演示系统。系统采用 Vue 2 + Element UI 实现纯前端 Mock 页面，用于根据各省份功能点评估表快速生成可点击、可演示、可验收的功能界面。

本项目不作为生产系统，不连接真实后端，不承诺真实数据持久化。验收关注点是功能点覆盖、页面可访问、按钮可点击、弹窗可展示、流程可闭环。

## 省份接入标准

每个省份以一个功能点评估 Excel 为主要输入。Excel 中应包含 `2、功能点拆分表` 工作表，并至少包含以下列：

- 客户需求
- 功能用户
- 功能用户需求
- 触发事件
- 功能过程
- 子过程描述
- 数据移动类型
- 数据组
- 数据属性
- 复用度
- CFP
- ΣCFP

标准映射关系如下：

| Excel 层级 | 系统表现 |
| --- | --- |
| 客户需求 | 一级业务分类或页面分组 |
| 功能用户 | 省份模块下的二级菜单 |
| 功能用户需求 | 具体演示页面 |
| 触发事件/功能过程 | 页面按钮、表格操作或流程节点 |
| 子过程描述 | 弹窗步骤、接口日志、详情说明或流程记录 |
| 数据组/数据属性 | 表格字段、表单字段、详情字段 |

## 命名标准

省份模块使用以下格式：

```text
{省份简称}{省份名称}COSMIC
```

示例：

- `JL吉林COSMIC`
- `YN云南COSMIC`

页面目录建议放在：

```text
src/views/资产信息上报调整/{省份模块}/{功能用户需求}/index.vue
```

路由菜单建议挂载到：

```text
资产信息上报调整 / {省份模块} / {功能用户} / {功能用户需求}
```

## 页面标准

每个功能用户需求页面默认包含：

- 查询区：关键字、状态、时间、数据来源、责任部门等。
- 工具栏：新增、导入、导出、批量处理、刷新等通用按钮。
- 数据表格：根据数据组和数据属性生成模拟字段。
- 操作列：详情、编辑、删除、审核、下发、反馈、日志等。
- 功能点按钮区：根据功能过程生成按钮或流程入口。
- 弹窗区：根据按钮类型展示表单、详情、审批、接口调用、稽核结果、统计图表等。

## 复杂页面聚合标准

当某个功能用户需求下的触发事件较多时，不建议将所有功能点平铺为按钮。推荐按业务对象聚合：

- 主页面保留核心流程入口，例如监控、告警、稽核、统计、采集、预处理、导出、推送。
- 新增、修改、删除、查询、上传、报告详情等围绕同一业务对象的功能，生成独立管理子页面承接。
- 主页面按钮数量建议控制在 20 到 40 个之间，超过后应继续拆分子页面。
- 验收时按“功能簇”说明覆盖关系，不要求按钮名与 Excel 功能点逐字一一对应。

参考样板：`docs/样板页面-4A上报数据对比告警规则管理.md`。

## Mock 数据标准

Mock 数据应满足演示需要：

- 每页至少生成 5 到 10 条演示数据。
- 状态字段应覆盖待处理、处理中、已完成、异常、已驳回等常见状态。
- 时间字段使用最近 30 天内的模拟时间。
- 编号字段使用稳定前缀，例如 `YN-2024-0001`。
- 数据内容应尽量贴合功能名称，不使用明显无关的占位文本。

## 可手改与不可手改

优先手工维护：

- README
- docs 下的规范、演示说明、验收清单
- 生成脚本
- 通用模板

谨慎手工修改：

- 自动生成页面
- 自动生成路由
- 批量生成的数据文件

如果自动生成页面存在共性问题，应优先修改生成脚本或页面模板，再重新生成。

## 验收标准

每个省份模块至少满足：

- 菜单层级完整。
- 页面可打开，无空白页。
- 查询、重置、刷新有反馈。
- 新增、编辑、详情、删除等基础按钮可点击。
- 导入、导出有演示反馈。
- 审批、下发、反馈、稽核、统计类按钮能打开对应弹窗。
- 构建命令 `npm run build` 通过。
- 已提供省份演示说明或验收清单。
