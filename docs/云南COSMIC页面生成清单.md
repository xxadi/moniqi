## 当前生成口径

当前脚本使用 `功能点拆分表.json` 作为输入。该 JSON 保留了：

- 客户需求
- 功能用户需求
- 触发事件
- 子过程描述
- 数据属性

当前 JSON 未保留 `功能用户` 字段，因此现有路由结构为：

```text
资产信息上报调整
└── YN云南COSMIC
    └── {功能用户需求页面}
```

标准化后的推荐结构为：

```text
资产信息上报调整
└── YN云南COSMIC
    └── {功能用户}
        └── {功能用户需求页面}
```

后续如需按 `功能用户` 二级分组，应从原始 Excel 重新解析，或扩展 `功能点拆分表.json`，保留 `功能用户` 字段。

## 页面生成规则

每个 `功能用户需求` 生成一个页面：

```text
src/views/资产信息上报调整/YN云南COSMIC/{功能用户需求}/index.vue
```

每个页面包含：

- 查询区
- 操作工具栏
- 演示数据表格
- 功能过程按钮区
- 详情/表单/审批/接口/稽核/统计等弹窗
- 导入组件 `ImportDialog.vue`

每个 `触发事件` 生成一个按钮。按钮点击后根据关键词打开对应弹窗或执行模拟操作。

每条 `子过程描述` 用于生成弹窗说明、流程步骤、处理日志或详情内容。

## 重新生成步骤

执行前建议先确认当前工作区改动，避免覆盖手工修改。

```bash
node scripts/generate-yn-pages.js
```

脚本会：

1. 读取 `功能点拆分表.json`。
2. 按 `功能用户需求` 分组生成页面。
3. 为每个页面生成触发事件按钮。
4. 为每个页面补充 `ImportDialog.vue`。
5. 更新 `src/router/modules/examples.js` 中的 `YN云南COSMIC` 路由块。

## 注意事项

- 当前脚本不会覆盖已存在的 `index.vue`，已有页面会跳过。
- 当前脚本会重写 `YN云南COSMIC` 路由块。
- 如需统一调整页面样式或按钮行为，优先修改 `scripts/generate-yn-pages.js`，再重新生成。
- 如需按功能用户分组菜单，应先补充数据源中的 `功能用户` 字段。
- 自动生成页面不建议逐个手工改；如果必须手工改，应在演示清单中记录原因。
